## 一、项目愿景

零门槛、零部署、零配置。打开浏览器即可使用 AI 编程助手。微信浏览器也能运行。分享链接即可传播。

### 目标用户

| 用户类型 | 场景 | 价值 |
|----------|------|------|
| **个人用户** | 日常 AI 助手 | 无需安装，打开即用 |
| **开发者** | 快速原型验证 | 复用 TrueConsole 验证过的模式 |
| **企业** | 内部工具分享 | 分享链接即用 |
| **微信用户** | 内置浏览器 | 微信生态无缝集成 |

## 二、可行性分析

### 浏览器能力

| 能力 | API | 支持度 |
|------|-----|--------|
| **存储** | IndexedDB / OPFS | 所有现代浏览器 |
| **文件操作** | File System Access API | Chrome / Edge |
| **HTTP 请求** | fetch + CORS | 需代理 |
| **后台运行** | Service Worker | Chrome / Edge |
| **本地推理** | WebLLM / Transformers.js | Chrome / Edge |
| **加密** | Web Crypto API | 所有浏览器 |

### 微信浏览器

支持：ES6+、IndexedDB、Service Worker（有限）、Web Crypto、fetch API。
不支持：File System Access API、WebGPU、WebRTC P2P。

**结论**：核心功能全平台可行，高级功能优雅降级。

### 开源项目验证

| 项目 | Stars | 证明 |
|------|-------|------|
| WebLLM | 17.6k | 浏览器内 LLM 推理可行 |
| Transformers.js | 13k+ | ML 模型可在浏览器运行 |
| Cherry Studio | 41.6k | 浏览器端 AI Agent 可行 |

## 三、核心挑战与解决方案

### CORS 限制

浏览器无法直接调用 LLM API（跨域限制）。

- **开发阶段**：公共 CORS 代理（如 allorigins）
- **生产环境**：Cloudflare Workers 反向代理
- **企业部署**：自建代理端点
- **用户层面**：设置中可配置自定义代理地址

### 文件系统

浏览器无法直接访问本地文件系统。

- **主方案**：基于 IndexedDB 的虚拟文件系统
- **增强**：File System Access API（Chrome/Edge）
- **降级**：`<input type="file">` 文件上传
- **导出**：程序化触发 `<a download>` 下载

### Shell 命令

浏览器无法执行真实 Shell 命令。

- 内置命令解释器，将 `ls`、`cat`、`grep` 等映射到 VirtualFS 操作
- 通用计算在沙箱 JavaScript 环境中执行

```

│                    JetBot（浏览器）                        │

│  界面层          React + Tailwind CSS                    │
│  ├─ 对话 / 宇宙视图 / 预览面板                            │
│  └─ 设置 / 权限 / 任务管理                                │
├─────────────────────────────────────────────────────────┤
│  Agent 层       （受 TrueConsole 启发）                   │
│  ├─ 工具循环     （熔断器 + 重复检测）                      │
│  ├─ 上下文管理   （滑动窗口）                               │
│  ├─ 技能注册     （18 个内置技能）                          │
│  └─ 计划模式     （观 → 谋 → 行 → 验）                     │
├─────────────────────────────────────────────────────────┤
│  工具层          10 个内置工具                             │
│  ├─ VirtualFS   （IndexedDB）                             │
│  ├─ HTTP / 搜索  （CORS 代理回退）                         │
│  └─ JS 执行 / HTML 渲染 / Shell                           │
├─────────────────────────────────────────────────────────┤
│  LLM 层          OpenAI 兼容协议                          │
│  ├─ 远程：OpenAI / DeepSeek / 自定义                      │
│  └─ 本地：Ollama                                         │
├─────────────────────────────────────────────────────────┤
│  存储层                                                   │
│  ├─ IndexedDB   （文件 + 定时任务）                        │
│  └─ localStorage（配置 + 偏好）                            │
└─────────────────────────────────────────────────────────┘
```

## 七、实施阶段

### 第一阶段：MVP

- 项目脚手架（Vite + React + TypeScript + Tailwind）
- Agent 核心（工具循环、上下文管理、熔断器）
- 虚拟文件系统（IndexedDB）
- LLM 客户端（OpenAI 兼容）
- 内置工具（读/写/搜索/执行）
- 基础界面（对话、输入、设置）

### 第二阶段：功能完善

- 计划模式（观 → 谋 → 行 → 验）
- 技能系统（18 个内置技能）
- 多服务商支持（OpenAI、DeepSeek、Ollama）
- 权限体系（三级）
- 定时调度（interval / cron / once）
- 文件桥接（拖拽导入 + 下载导出）

### 第三阶段：宇宙视图与未来

- 宇宙视图（力引导对话可视化）
- 跨轮次连线 + 断链控制
- 运行时检测（19 项浏览器能力）
- 微信浏览器优化
- 生态扩展
