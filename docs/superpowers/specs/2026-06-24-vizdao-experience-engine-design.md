# ViZDao 重构纲领：体验引擎（让知识被经历，而非被陈列）

> 状态：设计提案（待用户复核）· 日期：2026-06-24 · 方法论：道器合一「观→谋」
>
> 这是项目的**灵魂文件**。一切单元、视觉、交互，皆以此为尺。
> 一句话立意：**「我建了一个模型，它好不好？—— 看一眼就知道。」**
> 一条铁律：**深逻辑先于美学；美学是逻辑的诚实结果（Weniger, aber besser）。**

## 0. 初心与偏离

初心（见 [[vizdao-positioning]]、立项对话）：为行业继续教育学员做一台**零安装的实操训练台**——围绕一个聚焦目标，对一份真实数据**层层深入**，在**思考·操作·体验**中掌握分析套路。

偏离诊断（三处塌缩）：
1. **时间塌缩**：讲·做·悟被压成一张同屏静态面板，**悬念→揭示**的弧线消失，"抖包袱"抖不响（安斯库姆把"统计量相同"与"形状不同"同屏并列＝开演剧透）。
2. **主体塌缩**：学员是**观众**而非**主角**，缺了实操的支点——**预测·承诺·对峙**。拖一下看图变，是旁观现象，没有"我"的赌注，必然走马观花。
3. **主题塌缩**：做成了七个互不相干的**算法 demo 巡展**，而非**一条围绕真实数据的调查**。技术成了展品，而非"为回答问题而登场的工具"。

本纲领修正这三处塌缩。**算法弧线保留**；以 **Bike-Sharing** 真实数据为脊把七岛串成一条河；税务数据集后续接入。

## 1. 核心语法：设·赌·揭·悟·行

单元不是"带旋钮的图"，而是**一段有戏剧弧线的经历**。把**时间**与**门（gating）**变成一等公民：

```
设 Frame    立一个场景/疑问，一屏一事
赌 Predict  学员必须先承诺（选 / 估 / 拖一笔），才解锁下一拍   ← 实操的支点
揭 Reveal   抖包袱：编排好的揭晓，并回指学员的赌注（"你刚赌了 X"）
悟 Reflect  一句话沉淀（可接 AI 助教）
行 Explore  此时才放开旋钮——操作有了重量（去破坏 / 验证刚才的领悟）
```

这五拍是**语法**，不是建议。每个单元是这套语法的一次具体造句。"知识在'我以为'与'竟然是'的落差里长出来。"

## 2. 引擎架构（深逻辑：四个清晰边界）

把"怎么看/怎么动"和"何时发生什么"彻底分开。四个单一职责的单位：

```
Stage   舞台：一个贯穿全单元、不重建只变形的可视化surface。
        持有数据+当前视觉态；对外只暴露"指令(Directive)"，自己负责全部渲染与平滑过渡。
        例：showScatter('IV') / dropRegLine() / enableDrag('p8') / morph('quartetGrid')
Beat    一拍：脚本里的一步。声明 {kind, 旁白, 进入时触发的 Stage 指令, 推进条件}。
        Beat 永不碰像素，只发"意图"。
Ledger  账本：本单元记下学员的每次承诺（预测），供后续 Reveal 个性化回指。
Player  放映机（通用）：渲染当前拍的旁白+恰当的交互件(继续/预测控件/旋钮)，
        进入时向 Stage 发指令，监听推进条件，推进。一次只现一拍；Stage 在其下持续变形。
```

接口骨架（实现时落为 TS）：

```ts
interface Unit { id: string; title: string; stage: StageComponent; beats: Beat[] }

type Beat =
  | { kind: 'frame';   say: string;                         enter?: Directive[] }
  | { kind: 'predict'; say: string; commit: Commitment;     enter?: Directive[] }
  | { kind: 'reveal';  say: (l: Ledger) => string;          enter: Directive[] }
  | { kind: 'reflect'; say: string; aiPrompt?: string }
  | { kind: 'explore'; say: string; knobs: Knob[]; goal?: Checkpoint };

interface Commitment { id: string; kind: 'choice' | 'number' | 'point'; options?: {value:string;label:string}[] }
// Directive 词表是"每单元自有"的（安斯库姆有"落回归线"，过拟合有"升阶数"）。
// 通用的是 Player/Beat/Ledger；每单元自带一个 Stage 组件 + 一段 beats 脚本。
```

**为什么这套逻辑"自动"满足美学**（形式是逻辑的结果）：
- **一屏一事** ← Player 一次只现一拍。
- **可视化是主角、chrome 退场** ← Stage 是持续 surface，每拍至多叠加一句旁白 + 一个控件。
- **动效只为揭示** ← 变形是 Reveal 拍的显式指令；其余拍近乎静止。
- **学员是主角** ← Predict 拍把"门"交给学员；Ledger 让揭晓对他个人成立。
- **连续的电影感** ← Stage 不重建只变形，跨拍平滑过渡是引擎天生能力，而非每单元手搓。

每个单位都可独立理解与测试：Player/Ledger/推进逻辑是纯函数；Stage 是受指令驱动的渲染件。

## 3. 美学即令牌（从原则推导，不外贴）

Dieter Rams「尽可能少的设计」落成一套**极少且固定**的令牌，全站统一，像一件博朗电器：

- **字 Type**：一套字阶（模数比 ~1.25），旁白大而安静、行高从容、字行宽约 60–72 字符。排版承担叙事。
- **空 Space**：基线网格；Stage 全幅出血；留白是默认态，不是点缀。
- **色 Color**：近单色基底 + **唯一强调色**（"此刻该看的东西"）；分类色板仅在数据要求时登场（如类别着色），且克制。数据墨水压过装饰墨水。
- **动 Motion**：一条缓动曲线 + 三档时长——`即时 / 快(200ms) / 揭示(600–900ms)`。**揭示**用长档；其余快或即时。
- **诚实 Honesty**：真数据、真数学，不糊弄。

令牌少到能一眼记住——这本身就是克制的证明。

## 4. 内容主线（一条河：Bike-Sharing 为脊）

- **冷开场 · 安斯库姆**：立全片论点「看见 > 计算」。它太经典，留作钩子。
- **转场**：「现在，用这双眼睛去查一份真数据——城市单车。」
- **Bike-Sharing 成为后续技术的共同舞台**，技术因问题登场（不是"今天讲过拟合"，而是"要搞清 X，于是撞上过拟合"）：

| 单元 | Bike-Sharing 上的真实问题 | 登场的技术 |
|------|--------------------------|-----------|
| 过拟合 | 租车量随气温/时段怎么变？阶数一高就荒腔走板 | 多项式回归 + 正则 |
| 分布 | 一天的租车量长什么形状？ | 直方图 / KDE |
| 高维困境 | 一天十几个特征，怎么一眼看全？ | 多维散点的困境 |
| 聚类 | 731 天能自动分出"通勤型/休闲型/恶劣天气型"吗？ | K-Means |
| 降维 | 把多特征的日子压成 2D，相似的日子聚团 | PCA / UMAP（t-SNE 手写数字解团降为彩蛋/对照） |
| 收束 | 把这趟调查丢给 AI 助教解读 | 可插拔助教 |

- **后续**：同一引擎接入**税务数据集**——换数据即得新课，引擎不动。

## 5. 第一个子项目范围（样板先行，谋定后动）

**子项目 1 = 体验引擎 + 安斯库姆样板单元。** 先把引擎与"手感+美感"锁死在一个端到端标杆上，用户亲手验证后再传播。

**安斯库姆的新造句**（验证"抖包袱"复活）：
1. 设：满屏只有四组**统计量**并排（大字、安静）。「均值、方差、相关系数——完全相同。」
2. 赌：「那画出来，应该也差不多吧？」按钮 **会一样 / 会不同**，必选其一才继续。
3. 揭：四张散点**逐一浮现**，形状天差地别；一拍静默；四条**回归线落下**，一模一样压在四种形状上。（包袱抖响。）
4. 悟：「你刚赌了'一样'。统计量为什么骗了你？」
5. 行：「组 IV——把那个离群点**拖走**，看回归线崩塌。」操作此刻有了重量。

**包含**：通用 Player/Beat/Ledger/推进逻辑；预测控件（choice）；安斯库姆 Stage（含 morph/落线/可拖拽）；一套设计令牌；把安斯库姆登记进导航替换旧版。
**不做（后续子项目）**：Bike-Sharing 数据接入与其余六单元重铸；税务数据集；旧单元在被替换前保持在线不动。

## 6. 与现有代码的关系

- **并行新建，逐个替换**：新引擎落在 `src/viz/engine/`（Player/Beat/Ledger/tokens）与 `src/viz/units/`（每单元 = Stage + beats）。`registry` 在某单元重生后切换其入口；其余仍指向旧 `lessons/*`，**线上始终可用**。
- **可复用**：ECharts 封装思路（`ChartCanvas`）可被 Stage 内部沿用；纯算法（`ridge/pca/tsne/kmeans/kde`）原样复用——它们是诚实的"道具"，与表现层无关，正应留存。
- **退役**：旧 `lessons/*` 单元随替换逐个退役；当前未提交的"字形/补间"实验作废（属于被重构掉的旧范式）。

## 7. 测试策略

- **纯逻辑优先**：Player 推进（Predict 未承诺则不解锁）、Ledger 记忆与回指、推进条件判定——纯函数单测。
- **Stage 冒烟**：指令→视觉态切换不崩（mock 渲染）。
- **体验回归**：样板单元的"一拍拍能走通"端到端冒烟。
- 关键路径必测；视觉细节交由浏览器实测（吸取教训：单测 mock 渲染层，覆盖不到真库/真画布）。
